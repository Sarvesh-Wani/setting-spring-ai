Why You're Getting This Error
The root cause is SSLHandshakeException: PKIX path building failed — Java's trust store doesn't trust the SSL certificate of your SMTP server (likely Gmail or a corporate/self-signed mail server).
This happens because:

Your JDK's cacerts keystore is missing the mail server's CA certificate — common when running inside Docker, a minimal JDK image, or a corporate network with a custom root CA.
Or your application.properties has wrong TLS settings — e.g., using mail.smtp.starttls.enable=true on port 465 (which expects SSL directly), or vice versa.
Or the certificate on the SMTP server is self-signed / expired.


Fix 1 — Correct your application.properties (most likely cause)
For Gmail, it should be exactly this:
propertiesspring.mail.host=smtp.gmail.com
spring.mail.port=587
spring.mail.username=your-email@gmail.com
spring.mail.password=your-app-password

spring.mail.properties.mail.smtp.auth=true
spring.mail.properties.mail.smtp.starttls.enable=true
spring.mail.properties.mail.smtp.starttls.required=true
spring.mail.properties.mail.smtp.ssl.trust=smtp.gmail.com

Use port 587 + STARTTLS, NOT port 465 with STARTTLS. Port 465 requires mail.smtp.ssl.enable=true instead.

Fix 2 — Trust the SMTP server's certificate explicitly
Add this to your properties as a quick fix:
propertiesspring.mail.properties.mail.smtp.ssl.trust=smtp.gmail.com
This tells Java to trust that host even if the cert chain isn't fully in cacerts.

Fix 3 — Use an App Password (Gmail)
If using Gmail, your normal password won't work. You need a Google App Password:

Google Account → Security → 2-Step Verification → App Passwords → Generate one for "Mail"

How to Use the Email Service Across the Project
Based on the stack trace showing MailService.sendEmail() and AuthController, here's a clean, reusable pattern for the whole project:
MailService.java — generic reusable service:
java@Service
@RequiredArgsConstructor
public class MailService {

    private final JavaMailSender mailSender;

    @Value("${spring.mail.username}")
    private String fromEmail;

    public void sendEmail(String to, String subject, String body) {
        SimpleMailMessage message = new SimpleMailMessage();
        message.setFrom(fromEmail);
        message.setTo(to);
        message.setSubject(subject);
        message.setText(body);
        mailSender.send(message);
    }

    // For HTML emails (e.g. OTP, appointment confirmation)
    public void sendHtmlEmail(String to, String subject, String htmlBody) throws MessagingException {
        MimeMessage message = mailSender.createMimeMessage();
        MimeMessageHelper helper = new MimeMessageHelper(message, true, "UTF-8");
        helper.setFrom(fromEmail);
        helper.setTo(to);
        helper.setSubject(subject);
        helper.setText(htmlBody, true); // true = isHtml
        mailSender.send(message);
    }
}

Pro tip — wrap calls in a try-catch so a mail failure doesn't crash the whole request:
javatry {
    mailService.sendEmail(user.getEmail(), subject, body);
} catch (MailException e) {
    log.error("Failed to send email to {}: {}", user.getEmail(), e.getMessage());
    // Don't rethrow — email failure shouldn't block the main flow
}
